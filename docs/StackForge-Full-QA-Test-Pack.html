StackForge — Full QA / UAT Test Pack

Product: StackForge (Enlight Labs) — AI infrastructure scaffold generator
Purpose: Exhaustive manual test scripts for final acceptance before handoff
Audience: QA queue / Manager review
Version: 2026-07-22 · Covers interview → plan → generate → ZIP → scaffold checks
Scope: Generator only (does not provision cloud resources)

How to use this pack
  1. Open StackForge → click New Project for every case (do not reuse an old chat).
  2. Paste the Opening prompt exactly.
  3. Answer the interview with the listed picks (or type the same text).
  4. Review the plan → Approve & Generate.
  5. Run Scaffold checks (terraform / hadolint / actionlint / helm as applicable).
  6. Download ZIP and verify the Expected ZIP signals.
  7. Mark Pass / Fail in the result table at the end of each section.

Golden rule after any deploy: always New Project. Old ZIPs never update themselves.

2. Global pass criteria (apply to every generate case)

  • Interview appears (or plan if answers already in prompt).
  • Plan includes an Implement stage / clear file list.
  • ZIP contains Terraform under terraform/, CI file matching choice, and a health stub (not a full app).
  • environments/*.tfvars match region, access, DB/redis flags, scale, and env count.
  • README mentions applied interview options (or honest “unsupported” notes when needed).
  • No wrong-cloud bleed (e.g. ECS ZIP must not be EKS/OKE/AKS).
  • No junk trees like container_app/ or random charts/nginx*.
  • Scaffold checks: prefer PASS; if FAIL, use Fix once and re-check — should improve or pass.

12. ZIP audit cheatsheet (manager / reviewer)

  1. Unzip → list top-level folders.
  2. Open all environments/*.tfvars — confirm region, access, DB/redis, scale, env names.
  3. Confirm CI file matches interview.
  4. Confirm runtime stub language.
  5. Skim README “Scaffold options notes”.
  6. Confirm no wrong orchestrator files (ECS vs EKS vs OKE vs AKS vs Cloud Run vs ACA).
  7. Optional: run local terraform init/validate if toolkit available — or use in-app Scaffold checks.
# Quick grep helpers (PowerShell), after Expand-Archive: Select-String -Path .\environments\*.tfvars -Pattern 'enable_redis|enable_database|desired_count|alb_internal|aws_region|region|location' Get-ChildItem -Recurse -File | Select-Object -ExpandProperty FullName
